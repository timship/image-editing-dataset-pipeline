# Image Editing Dataset Pipeline

---
## 📌 О проекте

Проект представляет собой pipeline для генерации датасета изображений с редактированием по текстовой инструкции.

Основная идея:

```
исходное изображение → prompt → отредактированное изображение
```

Pipeline автоматически:
- загружает изображения из датасета
- генерирует инструкцию редактирования
- применяет модель image-to-image
- сохраняет результат и метаданные


---
## 🧠 Используемые технологии

- Датасет: Hugging Face `beans`
- Библиотеки: `diffusers`, `transformers`, `datasets`, `torch`
- Формат хранения: изображения + JSON

---

## ⚙️ Архитектура решения

В проекте реализовано **два подхода**:

### 1️⃣ Локальный pipeline

Файл: `prepare_dataset.py`

- используется облегчённая модель:

```
stabilityai/sd-turbo
```
- подходит для запуска на CPU / слабой GPU
- обеспечивает стабильную генерацию изображений

---
### 2️⃣ Основной pipeline (модель из ТЗ)

Файл:  

```
notebooks/colab_pipeline.ipynb
```

Используется модель:
```
diffusers/sdxl-instructpix2pix-768
```
Запуск выполнен в Google Colab с GPU (Tesla T4, ~15GB VRAM)

---

## ⚠️ Почему использовался Colab

Попытка запуска моделей из ТЗ локально показала ограничения оборудования:

### Qwen-Image-Edit
- требует значительно больше видеопамяти
- не подходит для запуска на GTX 1650 Ti (4GB VRAM)

### SDXL InstructPix2Pix
- модель загружалась локально
- но при генерации:
  - появлялись чёрные изображения
  - возникали проблемы с precision (float16 / float32)

Пробовались:
- изменение параметров генерации
- отключение safety checker
- уменьшение размера изображений

→ стабильного результата добиться не удалось

## ✅ Принятое решение
