# Image Editing Dataset Pipeline

---
## 📌 Что это за проект

Это небольшой pipeline, который:

- берёт изображения из датасета
- генерирует для них текстовую инструкцию
- прогоняет через модель
- сохраняет результат и JSON

Идея:

```
исходное изображение → prompt → отредактированное изображение
```
---
## 🧠 Что использовала

- датасет: beans с Hugging Face
- модель: stabilityai/sd-turbo
- библиотека: diffusers

---

## 📂 Что получается на выходе
Картинки
```
data/raw/     — исходные изображения  
data/edited/  — отредактированные  
```

JSON
```
[
  {
    "source_image_id": "source_0001",
    "edit_prompt": "a realistic photo of a plant leaf with water droplets",
    "edited_image_id": "edited_0001"
  }
]
```

---
## ⚙️ Почему не были использованы модели из ТЗ

В задании были предложены:

 - Qwen-Image-Edit
 - SDXL InstructPix2Pix

Я попробовала с ними поработать, но:

Qwen
- очень тяжёлая модель
- не подходит для запуска на моём ноутбуке
SDXL InstructPix2Pix
- скачалась и запустилась
- но при работе на видеокарте давала чёрные изображения

Пробовала:

- менять параметры
- отключать safety checker
- уменьшать размер

→ результат тот же

---
## 🛠 Что сделала в итоге

Запустила модель на CPU вместо GPU.

Да, медленнее, но:

- изображения генерируются нормально
- pipeline полностью работает

И выбрала более лёгкую модель:

```
stabilityai/sd-turbo
```

---

## 💡 Нюанс

Иногда изображения получаются странными (артефакты) — это нормально для таких моделей, особенно при слабом железе.

Здесь важнее был сам pipeline, а не идеальное качество картинок.

---

## 🚀 Как запустить
pip install -r requirements.txt
python prepare_dataset.py

---

## ⚠️ Ограничения
генерация на CPU медленнее, чем на GPU
изображения могут содержать артефакты (особенность diffusion моделей)
